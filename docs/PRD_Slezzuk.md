# 슬쩍 PRD

- 제품명: 슬쩍
- 슬로건: 슬랙인 줄 알았는데, 슬쩍!
- 한 줄 소개: Slack처럼 보이는 화면에서 채용 공고를 탐색하고, 저장하고, 로컬 키워드로 매칭하는 이직 준비 웹 서비스
- 핵심 제약: 현재 MVP는 GPT API Key와 로컬 Codex를 사용하지 않는다.

## 1. 제품 배경

이번 대회의 주제는 "팀장님이 보면 한숨 나올 서비스 만들기"다.

팀장님이 가장 깊게 한숨 쉬는 순간은 팀원이 조용히 이직을 준비하고 있었다는 사실을 알게 되는 순간이다. `슬쩍`은 이 상황을 제품 콘셉트로 풀어낸다.

사용자는 회사에서 매일 쓰는 Slack을 보는 것처럼 자연스럽게 채용 공고를 확인한다. 겉으로는 업무 메신저지만, 실제로는 채용 공고 탐색, 관심 공고 저장, 지원 메모, 이력서 기반 로컬 매칭까지 가능한 이직 준비 도구다.

즉, `슬쩍`은 "회사에서 Slack 보는 척, 이직 준비를 슬쩍" 할 수 있게 만드는 서비스다.

## 2. 제품 목표

1. Slack과 거의 동일한 UI/UX로 사용자가 위화감 없이 채용 공고를 볼 수 있게 한다.
2. 직군별 채널에서 최신 채용 공고를 빠르게 탐색하게 한다.
3. 이모지, 나중에 보기, DM, 스레드 등 Slack의 익숙한 기능을 채용 공고 관리 방식으로 재해석한다.
4. 사용자의 이력서/포트폴리오와 공고를 로컬 키워드 매칭으로 비교해 지원 판단을 돕는다.
5. ngrok public URL로 다른 팀도 데모를 확인할 수 있게 한다.

## 3. 사용자 경험 원칙

- 겉모습은 Slack이어야 한다.
- 채용 서비스처럼 보이면 안 된다.
- 모든 기능은 Slack의 기존 사용 방식 안에서 자연스럽게 동작해야 한다.
- 사용자는 "공고를 관리한다"가 아니라 "메시지를 읽고, 이모지를 찍고, DM을 보낸다"고 느껴야 한다.
- 매칭 결과도 챗봇 답변처럼 튀지 않고 Slack 메시지나 스레드 안에 녹아들어야 한다.

## 4. 기술 방향

### 데이터 수집

잡코리아 웹 데이터를 크롤링해 채용 공고를 수집한다.

잡코리아 API는 현재 사용할 수 없다고 가정한다. 따라서 MVP는 크롤링 기반으로 구현하되, 안정성을 위해 수집 결과를 캐싱하고 요청 빈도를 제한한다.

크롤링이 불안정할 경우 사람인 API를 임시 데이터 소스로 사용할 수 있다. 다만 서비스의 메인 데이터 소스는 잡코리아로 유지하고, 추후 잡코리아 API 또는 제휴 데이터로 교체 가능한 구조로 만든다.

API 키, 계정, 비밀번호 등 인증 정보는 문서나 코드에 직접 기록하지 않고 환경변수로 관리한다.

### 분석 실행

현재 MVP는 GPT API와 로컬 Codex를 호출하지 않는다.

웹 앱은 사용자 요청을 서버로 보내고, 서버는 로컬 키워드 파서와 JobKorea 크롤링 결과를 이용해 검색 의도와 매칭 점수를 계산한다.

### 접속 방식

MVP는 현재 PC에서 웹 서버를 실행하고, ngrok 터널이 제공하는 public URL을 공유하는 방식을 기본으로 한다.

같은 Wi-Fi 또는 로컬 IP 접속을 전제로 하지 않는다.

## 5. 핵심 기능

### 5-1. 채널

채널은 직군별 채용 공고 피드다.

사용자는 Slack의 채널 목록을 보는 것처럼 직군별 공고 목록을 확인한다.

기본 채널 예시:

- `#PM`
- `#iOS 개발자`
- `#AI 개발자`
- `#웹 디자이너`

사용자는 미리 정의된 직군 리스트에서 새 채널을 추가할 수 있다. 채널을 추가하면 해당 직군과 관련된 채용 공고가 자동으로 모인다.

### 5-2. 공고 메시지

각 채용 공고는 Slack 채널에 올라온 하나의 메시지처럼 표시된다.

공고 메시지에는 최소한 다음 정보가 포함된다.

- 회사명
- 공고명
- 직군
- 근무지
- 경력 조건
- 마감일
- 원문 링크

톤 설정에 따라 공고 문구는 원문 그대로 보이거나, Slack 대화체처럼 변환되어 보인다.

### 5-3. 이모지 저장

사용자는 공고 메시지에 Slack 이모지를 찍어 공고를 저장한다.

지원 이모지:

- `👀` 관심 있음
- `⭐` 지원 후보
- `💰` 연봉 좋음

이모지는 단순 반응이 아니라 공고 분류 태그로 사용된다. 사용자가 찍은 이모지는 좌측 `나중에 보기` 탭에서 다시 모아볼 수 있다.

### 5-4. 나중에 보기

`나중에 보기`는 사용자가 이모지로 저장한 공고를 다시 확인하는 공간이다.

사용자는 아래 기준으로 공고를 필터링한다.

- 관심 있음
- 지원 후보
- 연봉 좋음

Slack의 저장 메시지처럼 보이지만, 실제로는 개인 지원 후보 리스트 역할을 한다.

### 5-5. DM

DM은 사용자가 데이터를 입력하고 AI 결과를 확인하는 공간이다.

DM 화면은 실제 Slack DM처럼 내가 보낸 메시지와 상대가 보낸 메시지가 함께 보이는 채팅 UI로 구성한다.

#### Analyze Job Posting

채용 URL을 분석하는 DM이다.

사용자가 잡코리아 공고 또는 회사 채용 페이지 URL을 입력하면, 시스템이 해당 페이지를 파싱하고 핵심 내용을 요약한다.

분석 결과는 하나의 공고 스레드로 생성된다.

표시 정보 예시:

- 회사명
- 공고명
- 주요 업무
- 지원 자격
- 우대 사항
- 자소서 문항
- 인재상
- 마감일

#### 공고별 DM 메모장

공고 메시지의 프로필을 클릭하면 해당 공고 전용 DM으로 이동한다.

이 DM은 특정 공고만을 위한 개인 메모장이다. 사용자는 자소서 초안, 면접 준비 내용, 지원 전략 등을 자유롭게 기록한다.

추후에는 이 메모와 업로드한 파일을 기반으로 AI가 자소서 초안을 작성할 수 있다.

#### Resume & Portfolio

이력서와 포트폴리오 텍스트를 저장하는 DM이다.

사용자는 이력서 요약, 포트폴리오/프로젝트 설명, 핵심 스킬, 선호 조건을 입력한다. 시스템은 해당 텍스트를 로컬 상태에 저장한다.

저장된 텍스트는 공고별 키워드 매칭에 사용된다.

현재 MVP에서는 GPT API와 로컬 Codex를 사용하지 않는다.

#### Search

자연어로 채용 공고를 검색하는 DM이다.

사용자는 원하는 공고 조건을 문장으로 입력한다.

예시:

- `LG전자 SW 개발자 채용 공고 보여줘`
- `서울 근무 가능한 AI 개발자 공고 찾아줘`
- `연봉 좋은 PM 공고 찾아줘`

AI는 문장에서 회사명, 직군, 지역, 조건 등을 추출하고 잡코리아 웹 데이터를 검색한다. 결과는 최대 10개까지 DM 메시지로 보여준다.

### 5-6. 스레드

공고를 클릭하면 Slack 스레드가 열리고 상세 정보가 표시된다.

스레드 우측에는 사용자와 공고의 로컬 매칭 결과가 표시된다.

로컬 매칭 결과 예시:

- 이 공고와 잘 맞는 사용자 경험
- 부족하거나 보완하면 좋은 역량
- 이력서에서 강조하면 좋은 포인트
- 포트폴리오에서 연결하면 좋은 프로젝트
- 예상 면접 질문

### 5-7. 톤 조절

설정 메뉴에서 공고 표시 톤을 조절할 수 있다.

이 기능의 목적은 채용 공고를 채용 서비스처럼 보이지 않게 만드는 것이다. 같은 공고라도 Slack에서 동료와 대화하는 것처럼 보이도록 문장을 변환한다.

톤은 3단계다.

1. `raw`
   - 크롤링한 원문을 최대한 그대로 보여준다.
2. `비즈니스`
   - 업무 대화처럼 정돈된 말투로 보여준다.
3. `친근`
   - 친한 동료가 알려주는 듯한 말투로 보여준다.

## 6. 주요 화면 구조

### 좌측 사이드바

- 워크스페이스 영역
- 채널 목록
- DM 목록
- 나중에 보기
- 설정

### 중앙 영역

- 선택한 채널의 공고 메시지
- 선택한 DM의 채팅 메시지
- 검색 결과 메시지

### 우측 영역

- 공고 스레드
- 공고 상세 정보
- 로컬 매칭 결과

## 7. MVP 범위

MVP에서 구현할 기능은 다음과 같다.

1. Slack형 기본 UI
2. 직군별 채널
3. 잡코리아 공고 크롤링 및 캐싱
4. 공고 메시지 표시
5. 이모지 저장
6. 나중에 보기
7. URL 분석 DM
8. 공고별 메모 DM
9. 이력서/포트폴리오 업로드 DM
10. 자연어 검색 DM
11. 로컬 키워드 기반 매칭
12. 톤 조절
13. ngrok public URL 공유

## 8. 추후 확장

1. Teams, 네이버웍스 등 다른 업무 메신저 UI로 확장
2. 공고별 메모 기반 AI 자소서 초안 작성
3. DM 인터뷰를 통한 사용자 경험 데이터 수집
4. 이모지와 메모 이력을 반영한 개인화 공고 추천
5. 잡코리아 API 또는 제휴 데이터 연동

## 9. 제품 한 줄 정의

슬쩍은 회사에서 Slack 보는 척하면서 채용 공고를 탐색하고, 지원 준비까지 할 수 있는 Slack형 AI 이직 준비 서비스다.
