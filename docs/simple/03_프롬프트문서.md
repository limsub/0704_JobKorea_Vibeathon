# 슬쩍 프롬프트 문서

## 1. 기본 원칙

AI는 한 번에 모든 걸 하지 않는다.

3단계로 나눈다.

```text
이력서/포트폴리오 -> User Profile
채용 공고 -> Job Profile
User Profile + Job Profile -> Match Result
```

응답은 반드시 JSON으로 받는다.

## 2. User Profile 프롬프트

사용 시점:

Resume & Portfolio DM에 사용자가 이력서/포트폴리오를 입력했을 때.

입력:

```text
사용자 이력서/포트폴리오 텍스트
```

출력:

```json
{
  "desired_roles": [],
  "desired_locations": [],
  "hard_skills": [],
  "soft_skills": [],
  "tools": [],
  "projects": [
    {
      "title": "",
      "role": "",
      "skills": [],
      "responsibilities": [],
      "achievements": []
    }
  ],
  "must_have": [],
  "nice_to_have": [],
  "deal_breakers": []
}
```

프롬프트:

```text
You are an AI career profile extractor.

Analyze the user's resume and portfolio.
Extract only information explicitly written or reasonably inferred from the text.

Return ONLY valid JSON.
Do not use markdown.
Do not hallucinate.
If unknown, return an empty array.

===== USER DOCUMENT =====
{{USER_DOCUMENT}}
```

## 3. Job Profile 프롬프트

사용 시점:

채용 공고를 크롤링하거나 URL을 분석했을 때.

입력:

```text
채용 공고 본문 텍스트
```

출력:

```json
{
  "company_name": "",
  "job_title": "",
  "location": "",
  "employment_type": "",
  "salary": "",
  "experience": "",
  "responsibilities": [],
  "required_skills": [],
  "preferred_skills": [],
  "tools": [],
  "culture_keywords": [],
  "benefits": [],
  "cover_letter_questions": [],
  "deadline": ""
}
```

프롬프트:

```text
You are an AI job posting analyzer.

Extract structured information from the recruitment posting.

Return ONLY valid JSON.
Do not use markdown.
Do not infer unsupported facts.
If unknown, return an empty string or empty array.

===== JOB POSTING =====
{{JOB_POSTING}}
```

## 4. Match Result 프롬프트

사용 시점:

사용자가 공고 스레드를 열었을 때.

입력:

```text
User Profile JSON
Job Profile JSON
```

출력:

```json
{
  "overall_score": 0,
  "role_fit": {
    "score": 0,
    "reason": ""
  },
  "skill_fit": {
    "score": 0,
    "reason": ""
  },
  "experience_fit": {
    "score": 0,
    "reason": ""
  },
  "preference_fit": {
    "score": 0,
    "reason": ""
  },
  "strengths": [],
  "missing_points": [],
  "resume_tips": [],
  "interview_questions": [],
  "summary": ""
}
```

프롬프트:

```text
You are an AI recruitment matching engine.

Compare the User Profile and Job Profile.
Use only the supplied data.

Scoring:
- Role Fit: 25
- Skill Fit: 30
- Experience Fit: 20
- Preference Fit: 15
- Culture Fit: 10

Return ONLY valid JSON.
Use Korean.
Do not hallucinate.
Explain every score briefly.

===== USER PROFILE =====
{{USER_PROFILE_JSON}}

===== JOB PROFILE =====
{{JOB_PROFILE_JSON}}
```

## 5. Search Intent 프롬프트

사용 시점:

Search DM에서 자연어 검색을 입력했을 때.

출력:

```json
{
  "company": "",
  "roles": [],
  "locations": [],
  "career": "",
  "keywords": [],
  "search_query": ""
}
```

프롬프트:

```text
You are a Korean job search intent parser.

Convert the user's request into search conditions.

Return ONLY valid JSON.
Do not use markdown.

===== USER REQUEST =====
{{MESSAGE}}
```

## 6. Tone Rewrite 프롬프트

사용 시점:

공고 메시지를 Slack스럽게 바꿀 때.

출력:

```json
{
  "message_title": "",
  "message_body": ""
}
```

프롬프트:

```text
Rewrite this job posting as a Slack message.

Tone:
- raw: keep it close to the original.
- business: professional Slack message.
- friendly: casual teammate-like Slack message.

Return ONLY valid JSON.
Do not change facts.
Use Korean.

===== TONE =====
{{TONE}}

===== JOB PROFILE =====
{{JOB_PROFILE_JSON}}
```
