# 슬쩍 간단 개발 문서

## 1. 오늘 구현 기준

오늘은 실제 서비스처럼 만들지 않는다.

가장 단순한 구조로 데모가 돌아가게 만든다.

```text
브라우저
  -> 로컬 Python 서버
  -> 로컬 키워드 파서
  -> 잡코리아 크롤링/파싱
```

외부 사람에게 보여줄 때는 Vercel 배포 URL을 공유한다. 로컬 데모만 `ngrok`으로 공개 링크처럼 공유한다.

## 2. 현재 코드 기준

현재 본 프로젝트 코드는 아래 폴더에 있다.

```text
main_project/
```

주요 파일:

```text
public/index.html   Slack형 UI
public/app.js       프론트 동작
server.py           API 서버, 크롤링, 로컬 매칭
api/index.py        Vercel 함수 엔트리포인트
vercel.json         Vercel 라우팅 설정
data/job_roles.json 직군 채널 카탈로그
data/state.json     로컬 저장 파일
public/styles.css   UI 스타일
```

## 3. 실행 방법

```bash
cd main_project
./scripts/start_server.sh
```

브라우저:

```text
http://localhost:5174
```

Vercel 배포:

```bash
vercel deploy
```

로컬 공개 데모 링크:

```bash
./scripts/start_ngrok.sh
```

터널 도구가 만들어준 `https://...` 링크를 발표/테스트용으로 공유한다.

## 4. GPT API / 로컬 Codex 미사용 기준

현재 MVP에서는 GPT API Key를 사용하지 않는다.

로컬 Codex도 호출하지 않는다.

검색 의도 파싱과 공고 매칭은 아래 방식으로 처리한다.

```text
사용자 문장
  -> 로컬 키워드 파서
  -> JobKorea 검색어 생성
  -> JobKorea 검색 결과 크롤링
  -> 프로필/공고 키워드 기반 로컬 매칭
```

주요 함수:

```text
local_search_intent(message)
local_recruiter_search(message)
local_match(job)
```

## 5. 오늘 저장 방식

로그인은 구현하지 않는다.

로컬 MVP 상태는 서버의 `data/state.json`에 저장한다. Vercel 배포 환경에서는 `/tmp/jobkorea-vibe-state/state.json`에 임시 저장한다.

저장 방식:

- 공고 데이터: JobKorea 크롤링 결과
- 직군 채널 목록: `data/job_roles.json`
- 채널 표시 상태: `data/state.json`
- 이력서/포트폴리오 텍스트: `data/state.json`
- 이모지/메모: `data/state.json`
- 로컬 매칭 결과: 요청 시 계산

ngrok으로 접속한 팀원은 같은 로컬 데모 상태를 본다.

주의:

- 서버가 켜져 있는 동안 같은 JSON 상태를 공유한다.
- 상태를 초기화하려면 `data/state.json`을 초기 구조로 되돌린다.
- Vercel의 임시 상태는 콜드 스타트나 재배포 후 초기화될 수 있다.
- 실서비스용은 아니다.

## 6. API 최소 구성

오늘 필요한 API는 최소한만 둔다.

### 공고 목록

```text
GET /api/jobs?channel=pm
```

### 채널 목록

```text
GET /api/channels
```

### 채널 변경

```text
POST /api/channels
```

### URL 파싱

```text
GET /api/parse?url=...
```

### 프로필 저장

```text
POST /api/profile
```

### 로컬 매칭

```text
POST /api/match
```

### 자연어 검색

```text
GET /api/search?q=...
```

## 7. 오늘 구현 우선순위

### 1순위

- 화면 이름을 `슬쩍`으로 변경
- 로컬 Codex 제거
- GPT API Key 사용하지 않기
- public tunnel 공유 스크립트 추가
- 직군 채널 추가/삭제 구현
- Resume & Portfolio DM에서 텍스트 저장
- 공고 클릭 시 로컬 매칭 결과 표시

### 2순위

- Analyze Job Posting DM에서 URL 분석
- Search DM에서 자연어 검색
- 나중에 보기 화면
- 톤 조절

### 3순위

- 채널 추가
- 파일 업로드
- Playwright 자동 검색

## 8. 오늘 하지 않을 것

- 회원가입/로그인
- 실제 DB 구축
- 실제 운영 서버 배포
- 완전한 사용자 권한 관리
- 결제/지원 기능
- 완벽한 크롤링 안정화
