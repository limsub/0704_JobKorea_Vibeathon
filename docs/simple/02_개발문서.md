# 슬쩍 간단 개발 문서

## 1. 오늘 구현 기준

오늘은 실제 서비스처럼 만들지 않는다.

가장 단순한 구조로 데모가 돌아가게 만든다.

```text
브라우저
  -> 로컬 Python 서버
  -> 로컬 키워드 파서
  -> 잡코리아 크롤링/파싱
```

외부 사람에게 보여줄 때는 Vercel 배포 URL을 공유한다. 로컬 데모만 `ngrok`으로 공개 링크처럼 공유한다.

## 2. 현재 코드 기준

현재 본 프로젝트 코드는 아래 폴더에 있다.

```text
main_project/
```

주요 파일:

```text
public/index.html   Slack형 UI
public/app.js       프론트 동작
server.py           API 서버, 크롤링, Slack 메시지 변환, AI/로컬 매칭
api/index.py        Vercel 함수 엔트리포인트
vercel.json         Vercel 라우팅 설정
data/job_roles.json 직군 채널 카탈로그
localStorage        브라우저별 개인 저장소
public/styles.css   UI 스타일
```

## 3. 실행 방법

```bash
cd main_project
./scripts/start_server.sh
```

브라우저:

```text
http://localhost:5174
```

Vercel 배포:

```bash
vercel deploy
```

로컬 공개 데모 링크:

```bash
./scripts/start_ngrok.sh
```

터널 도구가 만들어준 `https://...` 링크를 발표/테스트용으로 공유한다.

## 4. GPT API / 로컬 Codex 미사용 기준

현재 MVP에서는 GPT API Key를 사용하지 않는다.

로컬 Codex도 호출하지 않는다.

검색 의도 파싱과 공고 매칭은 아래 방식으로 처리한다.

```text
사용자 문장
  -> 로컬 키워드 파서
  -> JobKorea 검색어 생성
  -> JobKorea 검색 결과 크롤링
  -> PDF 분석 JSON이 있으면 AI 매칭 코멘트, 없으면 프로필/공고 키워드 기반 로컬 매칭
```

주요 함수:

```text
local_search_intent(message)
local_recruiter_search(message)
local_match(job)
```

## 5. 오늘 저장 방식

로그인은 구현하지 않는다.

개인 MVP 상태는 서버가 아니라 브라우저 `localStorage`에 저장한다.

저장 키:

```text
slezzuk_local_state_v1
```

저장 방식:

- 공고 데이터: JobKorea 크롤링 결과
- 직군 채널 목록: `data/job_roles.json`
- 채널 표시 상태: 브라우저 `localStorage`
- 사용자 추가 채널: 브라우저 `localStorage`
- 직접 파싱한 공고 URL: 브라우저 `localStorage`
- 이력서/포트폴리오 텍스트: 브라우저 `localStorage`
- PDF 분석 결과: 브라우저 `localStorage`
- 이모지/메모: 브라우저 `localStorage`
- 매칭 결과: PDF 분석 JSON이 있으면 OpenAI Responses API로 계산, 없으면 요청 시 로컬 계산

ngrok이나 Vercel로 같은 URL에 접속해도 각 브라우저의 개인 상태는 서로 보이지 않는다.

주의:

- 브라우저 저장소를 지우면 개인 상태도 삭제된다.
- 다른 PC, 다른 브라우저에서는 같은 개인 상태가 이어지지 않는다.
- 민감한 이력서 데이터를 여러 기기에서 안전하게 이어 쓰려면 로그인 + DB가 필요하다.

## 6. API 최소 구성

오늘 필요한 API는 최소한만 둔다.

### 공고 목록

```text
GET /api/jobs?channel=pm
```

### 채널 목록

```text
GET /api/channels
```

### 채널 변경

```text
POST /api/channels
```

### URL 파싱

```text
GET /api/parse?url=...
```

### 프로필 저장

```text
POST /api/profile
```

### AI/로컬 매칭

```text
POST /api/match
```

### 자연어 검색

```text
GET /api/search?q=...
```

## 7. 오늘 구현 우선순위

### 1순위

- 화면 이름을 `슬쩍`으로 변경
- 로컬 Codex 제거
- GPT API Key 사용하지 않기
- public tunnel 공유 스크립트 추가
- 직군 채널 추가/삭제 구현
- Resume & Portfolio DM에서 텍스트 저장
- 공고 클릭 시 Slack 스레드 코멘트와 AI/fallback 매칭 결과 표시

### 2순위

- Analyze Job Posting DM에서 URL 분석
- Search DM에서 자연어 검색
- 나중에 보기 화면
- 톤 조절

### 3순위

- 채널 추가
- 파일 업로드
- Playwright 자동 검색

## 8. 오늘 하지 않을 것

- 회원가입/로그인
- 실제 DB 구축
- 실제 운영 서버 배포
- 완전한 사용자 권한 관리
- 결제/지원 기능
- 완벽한 크롤링 안정화
