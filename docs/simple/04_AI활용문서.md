# 로컬 파서 활용 문서

현재 MVP에서는 GPT API Key와 로컬 Codex를 사용하지 않는다.

## 1. 검색 의도 파싱

사용자가 Search DM 또는 검색 봇에 문장을 입력하면 서버가 로컬 규칙으로 키워드를 추출한다.

예시:

```text
LG전자에서 SW 개발 직군 공고 보여줘
```

처리 흐름:

```text
문장 정리
  -> 회사명 후보 추출
  -> 직군 키워드 추출
  -> JobKorea 검색어 생성
  -> 검색 결과 크롤링
```

주요 함수:

```text
local_search_intent(message)
local_recruiter_search(message)
```

## 2. 공고 매칭

Resume & Portfolio DM에 저장된 텍스트와 공고 제목/회사/키워드를 비교한다.

매칭 점수는 키워드 hit 수를 기준으로 계산한다.

주요 함수:

```text
local_match(job)
```

## 3. 사용하지 않는 것

- `OPENAI_API_KEY`
- OpenAI API 호출
- 로컬 Codex CLI 호출
- Codex Desktop adapter

## 4. 확인 방법

검색 봇 DM에서 문장을 보내면 trace 카드에 아래 값이 표시된다.

- parser mode
- query
- company
- role
- crawl URL
- result count
