# 슬쩍 간단 기획서

## 1. 한 줄 소개

슬쩍은 Slack처럼 보이는 화면에서 채용 공고를 보고, 저장하고, AI로 나와의 적합도를 확인하는 데모용 웹 서비스다.

## 2. 핵심 컨셉

대회 주제는 "팀장님이 보면 한숨 나올 서비스 만들기"다.

팀장님이 한숨 쉬는 순간은 팀원이 회사에서 조용히 이직을 준비하고 있었다는 걸 알게 되는 순간이다. 슬쩍은 이 상황을 Slack UI로 풀어낸 서비스다.

겉으로는 Slack을 보는 것 같지만, 실제로는 채용 공고를 탐색하고 이력서 기반 로컬 매칭을 보는 서비스다.

## 3. 오늘 구현 목표

오늘 목표는 완성형 서비스가 아니라 발표 가능한 데모다.

따라서 로그인, 실제 운영 서버, 완전한 보안, 복잡한 DB는 하지 않는다. 대신 핵심 경험이 보이게 만든다.

## 4. 핵심 기능

### 채널

직군별 채용 공고를 Slack 채널처럼 보여준다.

기본 채널:

- `#PM`
- `#iOS 개발자`
- `#AI 개발자`
- `#웹 디자이너`

### 이모지 저장

공고 메시지에 이모지를 찍어 저장한다.

- `👀` 관심 있음
- `⭐` 지원 후보
- `💰` 연봉 좋음

### 나중에 보기

내가 이모지로 저장한 공고를 모아본다.

### Analyze Job Posting DM

채용 공고 URL을 넣으면 공고 내용을 파싱하고 AI가 구조화한다.

### Resume & Portfolio DM

내 이력서/포트폴리오 텍스트를 입력한다. AI가 User Profile JSON으로 변환한다.

### Search DM

자연어로 공고를 검색한다.

예시:

```text
LG전자 SW 개발자 공고 보여줘
```

### Thread

공고를 클릭하면 오른쪽 스레드에서 상세 정보와 로컬 매칭 결과를 보여준다.

### 톤 조절

공고 문장을 Slack 메시지처럼 바꾼다.

- raw
- 비즈니스
- 친근

## 5. 데모 범위

오늘 데모에서는 다음만 보여주면 된다.

1. Slack처럼 보이는 화면
2. 채널별 공고 리스트
3. 이모지 저장
4. 나중에 보기
5. 이력서/포트폴리오 입력
6. 공고 URL 분석
7. 로컬 매칭 결과
8. ngrok 같은 공개 링크로 접속

## 6. 발표 때 말할 추후 개선

- 실제 로그인 도입
- Supabase/Postgres DB 저장
- 파일 업로드 지원
- Playwright 기반 잡코리아 검색 자동화
- 여러 사용자 데이터 완전 분리
- 잡코리아 API 또는 제휴 데이터 연동
- Teams, 네이버웍스 UI 버전 확장
